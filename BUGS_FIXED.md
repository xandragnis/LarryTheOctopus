# Regression Registry — DO NOT REINTRODUCE

## 2026-06-17: Plaintext GitHub PAT embedded in origin remote URL
- **File:** `.git/config` (local, untracked — `[remote "origin"]` url)
- **Cause:** Origin was set to `https://<classic-PAT ghp_…>@github.com/xandragnis/LarryTheOctopus.git`, embedding a GitHub Personal Access Token in plaintext. Any process, backup, or shell-history entry that reads the repo config could harvest it.
- **Fix:** Stripped the token from the URL → `https://github.com/xandragnis/LarryTheOctopus.git`. Git now obtains credentials from the already-configured global credential helper (`!gh auth git-credential`), which reads the token from the OS keyring. Verified: `git credential fill` returns a credential with no inline token, `git ls-remote --heads origin` succeeds (exit 0), and the keyring token has `ADMIN` on the repo (push works).
- **Guard:** Never embed a token in a remote URL (`https://<token>@github.com/...`). Keep origin token-free and let `gh auth git-credential` (keyring) supply auth. If `git remote -v` ever shows a `ghp_`/`github_pat_` string in the URL, the leak has been reintroduced. The exposed classic token MUST be revoked at https://github.com/settings/tokens (it is a separate token from the keyring `github_pat_` that `gh` uses, so revoking it does not break `gh`).

## 2026-06-17: Game blank / "nothing starts" — Babel 8 broke the JSX transform
- **File:** `LarryTheOctopusUpdated_Enhanced.html` (Babel CDN `<script>`, ~line 43)
- **Cause:** Babel was loaded unversioned (`unpkg.com/@babel/standalone/babel.min.js`), so unpkg served the newest release. Babel **8.0.1** changed `@babel/preset-react`'s default `runtime` from `classic` to `automatic`. The automatic runtime compiles JSX into `import { jsx } from "react/jsx-runtime"`. Babel standalone injects compiled `text/babel` output as a **classic** (non-module) `<script>`, so the browser throws `SyntaxError: Cannot use import statement outside a module`, the game script never executes, React never mounts, and the page is blank on every browser. The game uses the classic global pattern (`const { useState } = React;`) + React 18 UMD globals, which only the classic runtime (`React.createElement`) supports. Confirmed via in-browser `Babel.transform`: default/`automatic` emit the `import`; `classic` emits `React.createElement`.
- **Fix:** Pinned Babel to v7 → `unpkg.com/@babel/standalone@7.29.7/babel.min.js`. v7's preset-react defaults to the classic runtime, so no `import` is emitted and it runs as a classic script. Added an inline guard comment above the tag. Verified in a real browser: start menu renders, 0 console errors, START GAME transitions to a live 488×648 canvas.
- **Guard:** Do NOT remove the Babel version pin (`@7.29.7`, or at minimum `@7`). An unversioned or `@8` Babel reintroduces the automatic runtime and blanks the page. After any edit, load the file in a browser and confirm the start menu renders with 0 console errors. (Same lesson the already-pinned `react@18` / `tone@14.8.49` deps follow.)
